# v0.75.0 部署步骤

## 1. 覆盖 GitHub 文件

将升级包解压后，把其中全部文件上传到现有 `zg-auto-erp11` 仓库根目录并覆盖同名文件。提交说明建议填写：

`Upgrade Z&G AUTO ERP to v0.75.0`

Vercel 会自动开始新部署。

## 2. 保留数据升级 Supabase

如果 v0.74.0 已经能登录并保存数据，只运行：

`supabase/migrations/003_v075_upgrade.sql`

该脚本不会清空客户、车辆、工单或账号。

全新 Supabase 项目才运行：

`supabase/migrations/002_v074_isolated_install.sql`

## 3. Vercel 环境变量

继续保留：

- `VITE_SUPABASE_URL`
- `VITE_SUPABASE_PUBLISHABLE_KEY`

Publishable Key 应以 `sb_publishable_` 开头。不要填写 Supabase Secret Key。

## 4. 智能功能的可选部署

在 Supabase Functions Secrets 中配置：

- AI：`OPENAI_API_KEY`，可选 `OPENAI_MODEL`
- 短信：`TWILIO_ACCOUNT_SID`、`TWILIO_AUTH_TOKEN`、`TWILIO_FROM_NUMBER`
- 付款：`STRIPE_SECRET_KEY`、`PAYMENT_SUCCESS_URL`、`PAYMENT_CANCEL_URL`

然后部署以下 Edge Functions：

- `zg-ai`
- `zg-notify`
- `zg-payment`

这些密钥只能放在 Supabase 服务器端，不能放进 Vercel 前端变量或 GitHub。

## 5. 上线前检查

1. 登录后左下角应显示 v0.75.0。
2. 新建客户和车辆，测试 VIN 识别。
3. 新建库存配件并入库。
4. 新建工单添加人工和配件，保存后核对库存减少。
5. 修改配件数量，核对库存差额。
6. 分别打印 Estimate、Repair Order、Invoice 和 Receipt。
7. 记录一笔收款和支出，核对首页汇总。
8. 使用手机测试车牌拍照 OCR 和语音输入。
