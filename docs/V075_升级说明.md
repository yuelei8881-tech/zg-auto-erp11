# Z&G AUTO ERP v0.75.0 升级说明

## 本次已完成

- 客户、公司、车队、司机与车辆完整档案
- 车牌、VIN、Unit Number、司机和电话全局搜索
- 工单人工明细、配件明细、税费、折扣、收款、欠款和毛利润
- 工单保存自动扣库存；修改、取消或删除自动按差额退回
- 采购入库与完整库存流水
- Estimate、Repair Order、Invoice、Receipt 四种 Letter 紧凑打印模板
- VIN 自动识别（NHTSA vPIC）
- 浏览器语音生成维修记录
- 本地 OCR 车牌识别
- AI 故障诊断、AI 照片分类、Twilio 短信、Stripe 在线付款的安全云函数入口

## 数据升级

v0.75.0 继续使用 `zg_erp_records` JSONB 记录，因此不会清空 v0.74.0 数据。旧版工单的汇总人工费和配件费会自动转换成兼容明细。

`supabase/migrations/003_v075_upgrade.sql` 只增加安全索引和说明，不删除表。

## 外部服务配置

以下功能需要在 Supabase Functions Secrets 中配置密钥，并部署相应云函数：

- `zg-ai`: `OPENAI_API_KEY`，可选 `OPENAI_MODEL`
- `zg-notify`: `TWILIO_ACCOUNT_SID`、`TWILIO_AUTH_TOKEN`、`TWILIO_FROM_NUMBER`
- `zg-payment`: `STRIPE_SECRET_KEY`、`PAYMENT_SUCCESS_URL`、`PAYMENT_CANCEL_URL`

密钥只能放在 Supabase 云函数中，不能放到 Vercel 前端环境变量或 GitHub。
